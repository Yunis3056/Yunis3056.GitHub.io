多表查询

内连接：查询的都是两个表的交集。
外连接：查询的一张表中有，而另一张表中没有的数据。

内连接 [INNER] JOIN
  • 隐式内连接：看不到 JOIN 关键字，条件使用 WHERE 指定
  • 显示内连接：使用 INNER JOIN … ON 语句, 可以省略 INNER
外连接
  • 左外连接：用左边表的记录去匹配右边表的记录，如果符合条件的则显示；否则，显示 NULL
    可以理解为：在内连接的基础上保证左表的数据全部显示。OUTER 可以省略
  • 右外连接：用右边表的记录去匹配左边表的记录，如果符合条件的则显示；否则，显示 NULL
    可以理解为：在内连接的基础上保证右表的数据全部显示

子查询

子查询的概念：

一个查询语句中出现了另一个查询语句，里面的查询语句就是子查询，外面的查询语句就是主查询。

子查询的结果是一个值的时候

子查询结果是多行单列的时候

子查询的结果是多行多列

子查询小结

  • 子查询结果只要是单列，则在 WHERE 后面作为条件

  • 子查询结果只要是多列，则在 FROM 后面作为表进行二次查询

运算符

SQL语句中常用的运算符有以下几种：

  • 算术运算符：

    包括加(+), 减(-), 乘(*), 除(/)和取模(%)等，用于执行基本的数学运算。

  • 比较运算符：

    包括等于(=), 不等于(<>)，大于(>)，小于(<)，大于等于(>=)，小于等于(<=)等，用于比较两个值的大小或是否相等。

  • 逻辑运算符：

    包括AND（与）, OR（或）和 NOT（非）等，用于连接条件并生成更复杂的查询条件。

  • 字符串运算符：

    包括连接符(||)和LIKE等，用于字符串的拼接和匹配操作。

  • 位运算符：

    包括按位与(&), 按位或(|), 按位异或(^)等，用于对二进制数进行操作。

DISTINCT语句：用于去重查询结果。

DCL 数据控制语言 (Data Control Language)

我们现在默认使用的都是 root 用户，超级管理员，拥有全部的权限。但是，一个公司里面的数据库服务器上面可能同时运行着很多个项目的数据库。所以，我们应该可以根据不同的项目建立不同的用户，分配不同的权限来管理和维护数据库。

权限
权限列表
  1. SELECT：允许用户查询（读取）表中的数据。
  2. INSERT：允许用户向表中插入新的数据行。
  3. UPDATE：允许用户修改表中的数据行。
  4. DELETE：允许用户删除表中的数据行。
  5. CREATE：允许用户创建新的数据库对象，如表、视图、索引等。
  6. DROP：允许用户删除数据库对象，如表、视图、索引等。
  7. ALTER：允许用户修改数据库对象的定义，如修改表结构、添加或删除列等。
  8. INDEX：允许用户创建索引，以加快对表的查询速度。
  9. REFERENCES：允许用户创建外键约束，以保证数据的完整性。
  10. EXECUTE：允许用户执行存储过程、函数、触发器等。
  11. ALL：允许用户拥有所有权限。

事务

理解：一条或者一组sql组成的一个执行单元，这个执行单元中的sql要么全部执行成功，要么全部执行失败。

事务中常见的问题：

问题名称 描述
脏读 一个事务读取了另一个事务未提交的数据，导致读取到不一致的数据。
不可重复读 一个事务在同一时刻多次读取同一行数据，但是在此期间另一个事务修改了该行数据，导致读取 到的数据不一致。
幻读 一个事务在同一时刻多次执行同一个查询，但是在此期间另一个事务插入了新的数据，导致读取到的数据集合不一致。
丢失更新 两个事务同时更新同一行数据，其中一个事务的更新会被另一个事务覆盖，导致更新操作丢失。

事务的四大隔离级别：

​ 隔离级别越高 性能越差 安全性越高。

  1. 读未提交（Read Uncommitted）：在这个级别中，一个事务可以读取另一个事务未提交的数据。这种级别会导致“脏读”问题，即读取到未提交的数据。
  2. 读已提交（Read Committed）：在这个级别中，一个事务只能读取另一个事务已经提交的数据。这种级别可以避免“脏读”问题，但是可能会出现“不可重复读”和“幻读”问题。
  3. 可重复读（Repeatable Read）：在这个级别中，一个事务在执行期间读取的所有数据都是一致的，即使其他事务在此期间对这些数据进行了更新。这种级别可以避免“不可重复读”问题，但是可能会出现“幻读”问题。
  4. 串行化（Serializable）：在这个级别中，所有的事务依次执行，就像是串行执行一样。这种级别可以避免所有的并发问题，但是会降低数据库的性能。
